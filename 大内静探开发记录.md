# 大内静探 (SilentParamQuery) 开发记录

## 项目概述

静默安装参数查询工具，用于批量部署时快速查找安装包的静默参数。

- **作者**: 技术人员，用于批量装机
- **主线版本**: C++ 版本 (cpp 目录)
- **当前版本**: v1.3

## 版本历史

突然想起静默参数扫描工具都是十年前的了，有不少新的类型无法识别，于是就找AI开始写程序。
程序并非万能的，仍然有很多程序检测不出来，这个不能怪我和AI，要怪DIE，因为程序是依托于这个国际顶尖识别软件的，他判断不了我们也无能为力了。而且我自己也发现有些检测虽然正确，但并未使用默认的静默安装参数，比如极域软件就被改成了/S，这类情况没有办法检测到，除非对程序进行逆向工程。
1.0以.net4.8下的c#编写，程序调用 diec.exe 完成查看类型，再配对静默参数，打包后的文件9.5M，可识别20种静默安装exe格式，用5大的工具打包成独立文件，也可以直接解压后使用。
1.2版改为MinGW-w64编译，仍然调用 diec.exe 完成查看类型，再配对静默参数，可识别29种静默安装exe格式，未打包成单文件，同学们自己打包吧。软件本身不再依赖.net平台，但DIE的 diec.exe可能会有VC依赖，已把文件放到附带的tools文件夹中，但尚未在PE下测试是否能正常工作。1.2版只需要一个exe就可以运行，其他的文件是源代码文件和辅助文件，下面的tools是die当前最新版本。
1.3版添加两个按钮，前一个是运行当前文件夹下所有程序，后一个运行当前文件夹下所有程序执行程序和脚本。主要用于装机时要装的那些一大堆软件，脚本主要指bat、cmd、reg、vbs，其中只能消除bat和cmd里的pause，reg做了regedit /s调用。
1.31添加处理7z压缩包功能，当点击“运行全部程序和脚本”后，连同当前文件夹下的压缩包一起处理，凡是当前文件夹下有7z压缩包，就解压到program文件夹下，并建立快捷方式，如果是32位就解压到programx86下面。


### v1.3 (2026-05-10)

**新增功能:**
- 批量运行功能：两个按钮
  - "运行全部执行程序" - 运行 exe/msi/appx/msix/msixbundle
  - "运行全部程序和脚本" - 额外包含 vbs/bat/cmd/reg
- 智能排序：
  - 优先运行：文件名含 vc/vcredist/microsoft/runtime/dotnet/directx/微软/运行库
  - 最后运行：文件名含 primo/极域
- 3路并行执行，带进度条和状态栏实时显示
- 自动使用静默参数运行识别到的安装包
- bat/cmd 处理：生成"静默"副本，替换 pause 为 "echo 暂停已消除"
- reg 处理：使用 regedit /s 静默导入
- vbs 处理：使用 wscript.exe 运行
- 嵌入 Tools.zip 资源，单文件运行，退出时清理临时目录
- 排除自身不被批量运行

**修复:**
- 静默安装命令中文乱码（scanner.cpp 编码统一为 UTF-8）

### v1.2

- 初始版本，29种安装器检测
- C++ 独立版本，无 .NET 依赖

## 构建说明

**环境要求:**
- MinGW-w64 14.2.0 (安装在 C:\mingw64)
- MSYS2 或 Git Bash

**构建命令:**
```bash
export PATH="/c/mingw64/bin:$PATH"
cd cpp
windres resource.rc -o resource.o
windres tools_resource.rc -o tools_resource.o
x86_64-w64-mingw32-g++ -static -O2 -mwindows -municode -std=c++17 \
  -DUNICODE -D_UNICODE -o SilentParamQuery.exe \
  main.cpp scanner.cpp batgen.cpp batchrun.cpp resource.o tools_resource.o \
  -lcomctl32 -lshell32 -lole32 -lgdi32 -lcomdlg32 -luser32 -lshlwapi
```

**输出:** SilentParamQuery.exe (~14MB，含嵌入的 Tools)

## 功能说明

### 主要功能
1. 拖放或浏览选择安装包文件
2. 自动识别安装器类型（29种）
3. 显示静默安装参数
4. 生成批处理脚本
5. 批量运行文件夹内所有程序

### 批量运行逻辑
- 启动时检查 exe 同目录是否有 Tools 文件夹
- 如果没有，从嵌入的资源解压到临时目录
- 扫描当前文件夹（不递归），按优先级排序
- 使用 CreateProcess 并行运行（最多3路）
- exe/msi 等自动使用检测到的静默参数
- 退出时清理临时文件

### UI 布局
- 文件选择区（拖放/浏览）
- 检测结果区
- 静默参数列表
- 静默命令显示 + 复制按钮
- 批处理生成选项
- 批量运行按钮
- 状态栏（左文字 + 右进度条）
